ASP.NET 5: Architecture and Features

  • Startseite
  • Blog
  • Technology
  • ASP.NET 5: Architecture and Features
  • teilen 
  • twittern 
  • merken 

The first preview release of ASP.NET 1.0 came out almost 15 years ago. Since then millions of developers have used it to build and run great web applications, and over the years Microsoft has added and evolved many, many capabilities to it. Lately, Microsoft introduced its new ASP.NET 5 platform which is a significant redesign of ASP.NET. Engineers at Microsoft were not even sure to use ASP.NET in the name at all! This topic introduces the new concepts and features in ASP.NET 5.

Introduction

ASP.NET 5 is a new fully open-source (see it on GitHub) to encourage community contributions and engagement, and it is cross-platform framework for building modern cloud-based Web applications including a unified story for building Web UI and Web APIs that integrate with today’s modern client-side frameworks and development workflows. Development team built it from the ground up to provide an optimized development framework for apps that are either deployed to the cloud or run on-premises.

It consists of modular components with minimal overhead, so you can add your dependencies packages, with built-in dependency injection support, based on your needs and retain flexibility while constructing your solutions. To appeal to a broader audience of developers, ASP.NET 5 supports cross-platform development where you can develop and run your applications on Windows, Mac and Linux.

It comes with a new, agile project system in Visual Studio while also providing a complete command-line interface so that you can develop using the tools of your choice.

Architecture

ASP.NET 5 has a number of architectural changes that make the core web framework much leaner and more modular. It is no longer based on System.Web.dll, but is instead based on a set of granular and well factored NuGet (package manager) packages allowing you to optimize your app to have just what you need. You can reduce the surface area of your application to improve security, reduce your servicing burden and also to improve performance in a true pay-for-what-you-use model.

ASP.NET 5 runs on the top of the existing .Net 4.6 framework which is the full-featured framework (you have always known) that runs over IIS/Windows, but it also has the capability to run on top of .Net Core which is the new lightweight cross-platform version of the .Net framework. It is lightweight because it brings its features on packages that you can manage based on your own application requirements which allows it to be much smaller and faster. It is cross-platform because of the CoreCLR and the self-hosted feature that does not required deployment of your ASP.NET 5 application on IIS explicitly.

.Net Core

.NET Core 5 is a modular runtime and library implementation that includes a subset of the .NET Framework. Currently it is feature complete on Windows, and in-progress builds exist for both Linux and OS X. .NET Core consists of a set of libraries, called “CoreFX”, and a small, optimized runtime, called “CoreCLR”. .NET Core is open-source, so you can follow progress on the project and contribute to it on GitHub:

  • .NET Core Libraries (CoreFX)
  • .NET Core Common Language Runtime (CoreCLR)

The CoreCLR runtime (Microsoft.CoreCLR) and CoreFX libraries are distributed via NuGet. The CoreFX libraries are factored as individual NuGet packages according to functionality, named “System.[module]” on nuget.org.

One of the key benefits of .NET Core is its portability. You can package and deploy the CoreCLR with your application, eliminating your application’s dependency on an installed version of .NET (e.g. .NET Framework on Windows). You can host multiple applications side-by-side using different versions of the CoreCLR, and upgrade them individually, rather than being forced to upgrade all of them simultaneously.

CoreFX has been built as a componentized set of libraries, each requiring the minimum set of library dependencies (e.g. System.Collections only depends on System.Runtime, not System.Xml). This approach enables minimal distributions of CoreFX libraries (just the ones you need) within an application, alongside CoreCLR. CoreFX includes collections, console access, diagnostics, IO, LINQ, JSON, XML, and regular expression support, just to name a few libraries. Another benefit of CoreFX is that it allows developers to target a single common set of libraries that are supported by multiple platforms.

Features

  • Totally Modular: You can include the needed packages independently like serving static files, MVC, logging, identity, …etc. Everything is optional.
  • Faster development cycle: with the self-compiled feature, you can save and run your app without having to compile it each time you made a change to your code. You can also use a rich set of code editors and tools that support ASP.NET 5 setup, code, build, and publish. For example, you can use VS code or Sublime over Linux or OSX with Yeoman tool to create your project template and OmniSharp tool that adds intelliSense for your code and many more additional features.


  • Cross-platform and open-source: ASP.NET 5 apps build over .Net Core can run on different operating systems like Win, Linux, and OSX. For the first time, all developers and designers can start building apps with it. And, they can use their favorite development environments such as Sublime Text and WebStorm when working with ASP.NET apps (No Visual Studio required). You also have access to the framework code which enables you to build the whole stack on your machine, or build your own packages and share them.
  • Self-hosted: ASP.NET 5 is completely decoupled from the web server environment that hosts the application. It supports hosting in IIS and IIS Express, and self-hosting scenarios using the Kestrel and WebListener HTTP servers.
  • Single platform: Where MVC and WebAPI are combined, and WebForms is gone.
  • Tag helpers: Tag helpers is the one feature that might have the biggest impact on the way that you create your views in an ASP.NET 5 MVC application. Tag Helpers are a better alternative to using traditional MVC helpers. They can be used to extend the HTML elements inside the View with server-side functionality.
  • Client side packages management support: Front-end development gets a lot of love in ASP.NET 5 through its support for GruntJS, NPM, and Bower. While you can use NuGet for server side packages management.
  • AngularJS: AngularJS is one of the most popular client-side frameworks for building Single Page Applications (SPAs). Visual Studio 2015 includes templates for creating AngularJS modules, controllers, directives, and factories. The support in ASP.NET 5 for GruntJS makes ASP.NET an excellent server-side framework for building client-side AngularJS apps. You can combine and minify all of your AngularJS files automatically whenever you perform a build. You can interact with an MVC 6 controller from an AngularJS $resource using REST.
  • net: In previous versions of ASP.NET MVC, the default testing framework was the Visual Studio Unit Testing Framework (sometimes called mstest). ASP.NET 5 uses xUnit.net as its unit test framework. This framework uses the [Fact] attribute instead of the [TestMethod] attribute (and no [TestClass] attribute]).

Some of the Popular ASP.NET Packages

  • Entity framework: Entity framework uses inverse engineering to build your database from entities classes. It is an object-relational mapper that enables .NET developers to work with relational and non-relational data using domain-specific objects. It eliminates the need for most of the data-access code that developers usually need to write.
  • Auto mapper: It helps in creating a mapping between a source and destination model types. Once the mapping is created then the source model object can be converted to a destination model object with ease and with less cluttered code. The developers can get rid of the explicit conversions code.
  • Identity: ASP.NET Identity is the new membership system for ASP.NET applications. ASP.NET Identity makes it really easy to customize profile and add Login/Logout functionality to the application.
